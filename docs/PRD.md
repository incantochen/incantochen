## 1. 目的與背景

### 1.1 文件目的
界定 MVP 要解決誰的什麼問題、做什麼／不做什麼、以及「成功」如何衡量，作為後續動線（P03）、資訊架構（P04）、線框（P05）與開發任務（T01+）的共同依據。

### 1.2 市場背景（取自 P01）
- **婚戒市場已飽和且品牌主導**：台灣婚戒消費高度集中於品牌（如品牌白鑽婚鑽），新進者在此正面競爭不利。
- **台灣高端／訂製站幾乎全是人工流程**：預約諮詢 → 設計師手繪／選鑽 → 人工報價 → 製作（常 1.5–2.5 個月）。線上多半只是型錄與預約入口，**極少有自助、即時報價、像一般電商一樣結帳的體驗**。
- **國際站已把自助配置器＋即時計價＋自助結帳做成標準**（James Allen、Blue Nile），但都是白鑽婚戒邏輯，且非台灣在地金物流／發票／語境。

### 1.3 機會
台灣存在一條空白帶：**自助、即時報價、以彩色寶石為主角、走標準電商結帳的高端半客製戒指**。本產品切入此空白，並刻意避開飽和的婚戒正面戰場。

## 2. 目標客群與 Persona

### 2.1 核心客群
**30–45 歲**、都會、經濟獨立的女性；**為自己購買**（self-purchase），而非等待被贈與或婚嫁採購。

關鍵心理輪廓（quiet luxury／小眾設計取向）：
- **注重設計與質感**、品味先行；喜歡**小眾設計品牌**，不追隨大 logo。
- **主導性格較強**：自己做決定、不需要被說服、討厭被推銷。
- 卡在一個明確的**價格／質感缺口**裡——「買品牌高端設計珠寶嫌太貴，買一般飾品又沒有高級感」。
- 風格**低調不張揚**，但要的是「**懂的人一眼看得出**」的非凡質感（戴在身上、配在包上的點綴，細看都有料）。

### 2.2 代表性 Persona — 「Vivian，38，設計事務所合夥人」
- **情境**：收入穩定、品味挑剔；對她而言珠寶不是收在保險箱的「資產」，而是**日常穿搭的一部分**——戴著它過日子、配出自己的風格。但市面選擇兩頭不對：大品牌高端設計珠寶覺得「為 logo 付太多、太招搖」，一般飾品又「一看就沒料、撐不起質感」。
- **想要**：低調但有質感、有設計感、**彩色寶石**為主角、**日常戴得住**的戒指；它要能延伸她嚮往的那種生活風格，要的是「懂的人看得出來」，不是「全場都看到牌子」。
- **購買觸發**：不是「我值得獎勵」這種里程碑敘事，而是**識別感**——當她覺得「**這件東西就是我**」，就會下手。
- **性格**：主導、果斷，自己研究、自己決定，**最反感被業務推銷**或被迫預約諮詢。
- **行為**：線上購物嫻熟、看重設計與細節、希望價格透明、能自己慢慢配、滿意就下單。偏好小眾設計品牌的調性。
- **痛點**：訂製站要預約／諮詢／等報價，門檻高又怕被推坑；品牌珠寶太貴又太張揚；平價飾品沒高級感。**中間這塊剛好沒人好好做。**
- **不是誰**：不是籌備婚禮的新人，不是等人送禮的對象，也不是追大牌 logo 的人。

### 2.3 客群洞察 → 產品含義
| 洞察 | 產品含義 |
|---|---|
| 卡在「品牌太貴太招搖」與「平價沒高級感」中間 | 定位明確填補此缺口：**設計感與質感對得起 NT$2–5 萬，但不為 logo 溢價** |
| 低調不張揚、要「懂的人看得出」 | 視覺與商品走 **quiet luxury**：細節到位、不浮誇；呈現要能傳達「細看有料」 |
| 主導、果斷、反感推銷 | 全程**自助、價格透明**，無強制諮詢；資訊充分讓她自己決定 |
| 喜歡小眾設計品牌 | 品牌調性走**設計師品牌感**，非大眾連鎖；敘事重設計理念與選石 |
| 珠寶＝日常穿搭、生活風格的一部分 | 首款設計需**日常戴得住**（舒適、耐看、好搭）；呈現須有**配戴／生活情境圖**，而非只有白底商品照 |
| 購買觸發＝「這件東西就是我」的識別感 | 文案與商品敘事走**身份認同／生活風格**（不是「妳值得獎勵」）；幫她在選配中投射出「這就是我」 |
| 彩寶＝個人風格表達 | 商品與配置器以**寶石顏色**為第一主角；可帶誕生石／象徵性敘事 |
| 線上購物熟練、重質感 | 體驗與視覺需精緻；RWD（多為手機）、載入順、呈現可信 |

## 3. 價值主張與定位

### 3.1 定位陳述
> **填補「品牌高端珠寶太貴太張揚」與「一般飾品沒有高級感」之間的空缺——台灣第一個自助、即時報價、以彩色寶石為主角的高端半客製戒指站，為注重設計與質感、主導性格、喜歡小眾設計品牌的女性，提供低調卻「懂的人一眼看得出」的質感選擇。**

### 3.2 價值主張（對 Persona）
> 妳的品味不該只能二選一：要嘛為大 logo 付過頭、要嘛將就沒料的平價飾品。我們做的是中間那塊——以彩色寶石為主角、日常戴得住、可線上自助選配、即時看到價格的高端半客製戒指。讓珠寶成為妳日常穿搭與生活風格的一部分；低調不張揚，但細看就有料。當妳覺得「這件東西就是我」，價格透明、下單即走，下單後為妳訂製。

### 3.3 差異化支柱
1. **填補價格／質感缺口**：質感對得起 NT$2–5 萬，但不為品牌 logo 溢價——大牌與平價之間的空白帶。
2. **彩色寶石為核心**（非白鑽）— 與品牌婚鑽市場區隔，也支援低調而有個性的風格表達。
3. **Quiet luxury／小眾設計調性**：低調、設計先行、「懂的人看得出」，非大眾連鎖的張揚感。
4. **自助 × 即時報價 × 透明**（非預約諮詢）— 對應主導、反感推銷的性格；與台灣訂製站區隔。
5. **半客製的可控訂製**：標準款 + 白名單選配，兼顧個性化與單人可營運。

## 4. 產品目標與 MVP 成功指標

### 4.1 MVP 階段定位
MVP 是「**會走路的骨架**」：單一戒指款、可配置、可付款、可出貨、可管理訂單。目的在**驗證可行性與體驗**，而非衝量。

### 4.2 成功指標（已拍板）
**主指標（North Star）— 完整閉環跑通：**
- 真實顧客能完成**端到端下單**：選配 → 報價 → 結帳（結帳即會員＋免密碼登入：Email OTP 為主、magic link 為輔）→ 綠界付款成功 → 開立電子發票 → 收到確認信 → 後台看得到單並可改狀態／貼單號 → 出貨通知。
- 驗收：至少數筆真實交易完整走完，無中斷、金額正確、發票與通知到位（**先看「能不能跑完」，不看量**）。

**輔助指標 — 配置器體驗有效：**
- **配置完成率**：進入配置器 → 完成選配（含寶石／金屬色／戒圍／數量）的比率。
- **配置 → 加入購物車轉換率**：完成選配 → 加入購物車的比率。
- 用途：判斷選配流程是否直覺、選項是否好懂、價格是否讓人安心。

### 4.3 觀測機制
- 以基本網站分析（GA4，T60）追蹤配置器漏斗事件（進入／選配完成／加購／結帳）。
- 上線初期以「能否跑通＋漏斗有無明顯斷點」為判讀重點；**量化目標值待有 baseline 後再設**，避免上線前憑空訂 KPI。

### 4.4 明確的非目標（MVP 不追求）
- 不追求成交量／GMV 規模；不做成長行銷投放優化；不做多品類廣度；不做 3D 即時預覽。

## 5. 範圍界定

### 5.1 MVP 做（戒指優先）
- 會員：結帳即會員 ＋ 免密碼登入（Email OTP 為主、magic link 為輔）。
- 戒指商品詳情頁 ＋ **資料驅動配置器**（寶石／金屬色／戒圍／數量，白名單控制）。
- **報價引擎**：底價 ＋ Σ選項加價 × 數量，即時計算、**伺服器端為準**。
- 規格＋金額**快照**（加入購物車／成立訂單當下釘住價格與規格）。
- 購物車、結帳（收件＋配送）、**綠界金流**、**電子發票**。
- **黑貓宅配**（保價＋本人簽收，不開放超商）。
- 訂單狀態與通知（下單確認、出貨通知；新單通知店家）。
- 後台訂單管理（看單／改狀態／貼物流單號／PII 遮罩與稽核）。
- 回頭補的後台商品 CRUD（款式／選項／白名單／圖片）。
- 上線法規配套（隱私／服務條款、客製例外告知與同意、賣家資訊揭露、個資權利處理流程）。
- 商品**組合圖：靜態疊圖**（Blender 3D 素材 ＋ 前端擬真疊圖；選項變動換圖）。

### 5.2 MVP 不做（Phase 2+ / 後續）
- **3D 即時預覽**（改用靜態疊圖）、LINE 通知、工單系統、品管模組、物流自動追蹤（手動貼單號）。
- **其他品類**（耳環／項鍊／手鍊）——靠後台自行擴充。
- **全客製**（報價→確認書→鎖價）為 Phase 3。

> 範圍與既有 memory §4 一致；本 PRD 不擴張範圍，僅補上客群／定位的依據。

## 6. 核心使用者動線（概述，細節見 P03）

1. **首購下單**：發現戒指 → 進配置器選配（即時看價）→ 加入購物車 → 結帳填收件（結帳即會員）→ 同意客製例外條款 → 綠界付款 → 成功頁 → 收確認信。
2. **回訪查單**：免密碼登入（OTP／magic link）→ 會員中心 → 看訂單狀態與紀錄。
3. **售後申請**：瑕疵／錯誤 → 申請 → 審核 → 綠界退刷 → 狀態與通知（七天鑑賞期：標準品適用，客製品主張法定例外、瑕疵仍可退；**用詞待律師審**）。

## 7. 功能需求（PRD 層級，對應任務模組）

### 7.1 商品與配置器
- 戒指商品詳情頁，呈現選配後的靜態合成圖（正面／側面）＋**配戴與生活情境圖**（戴在手上、融入日常穿搭的樣子）——讓客群投射「這就是我」，而非只看白底商品照。
- 配置器：分區漸進式選擇（**寶石顏色為第一主角**）、即時總價、加價明細可展開。
- 選項相容性由**三層白名單**控制（`applies_to` → `ProductOption` → `ProductOptionValue`），前端不得繞過。
- 戒圍對照／量法說明（降低選錯尺寸的退貨風險）。

### 7.2 報價與快照
- 報價＝底價 ＋ Σ選項加價 × 數量；**伺服器端重算為準**，前端僅顯示。
- 加入購物車／成立訂單寫入 `unit_price_snapshot` ＋ `config_snapshot`(JSON)；後台日後調價**不得影響已成立訂單**。

### 7.3 會員與結帳
- 免密碼登入（Email OTP 為主、magic link 為輔）；結帳即建會員（以 email 辨識既有／新會員，會員 id ＝ auth.uid()）。
- 結帳填收件與配送；**內嵌客製例外告知＋勾選同意（存同意內容與時間戳）**。

### 7.4 金流、發票、物流
- 綠界付款請求／導頁／回拋驗證／狀態更新；**冪等性**處理防重複。
- 付款成功後開立電子發票（綠界）。
- 黑貓宅配（保價＋本人簽收）；**僅收台灣宅配費，國際段內含定價**（決策 #5）。

### 7.5 訂單、通知、後台
- 訂單狀態機 ＋ 狀態紀錄；顧客於會員中心追蹤。
- 通知：下單確認、出貨等狀態 email；新單通知店家。
- 後台：訂單管理、改狀態、貼單號；**PII 遮罩＋存取稽核**。
- 後台商品 CRUD（回頭補）：款式、選項類型／值、款式選項對應＋白名單、圖片上傳。

### 7.6 內容與呈現
- 靜態疊圖機制（對齊＋陰影高光）＋選項即時換圖；3D 素材以 Blender 自製。

## 8. 非功能需求

- **安全（最高優先）**：伺服器端驗價（紅線）、資料庫備份（紅線）、RLS、卡號不落地、最小化蒐集（不做欄位級加密）、應用層防護（Zod 驗證／SQL 注入防護／安全標頭／XSS／OTP·magic link 防濫發）、框架跑最新修補版。
- **法規**：隱私權政策／個資告知、客製例外告知與同意、賣家資訊揭露與聯絡頁、當事人個資權利處理流程；**法律用詞以律師審定版為準**。
- **效能／體驗**：RWD（手機優先）、圖片效能優化（lazy load、Next.js Image）、載入／過場／空狀態處理、客製錯誤頁。
- **SEO／分析**：meta／OG／sitemap／robots／商品結構化資料；GA4＋轉換追蹤＋Cookie 同意。
- **可信度**：規格透明（每選項標材質／成色）、交期明示、呈現品質驗收（對齊精準、光影自然、與實品一致）。

## 9. 商業／定價參數

- **目標客單價帶**：中段 **NT$20,000–50,000／枚**（含稅前）。屬「輕奢自用」區間，材質與寶石選擇需落在此帶可達成的成本結構。
- **定價邏輯**：底價 ＋ 選項加價（含**國際物流成本緩衝**）；金屬價格波動於上架定價時以成本緩衝吸收（決策 #1）。
- **運費**：台灣宅配段向客人收；國際段（海外工廠→台灣）內含商品定價、不另收（決策 #5）。
- **退款**：客製品限瑕疵／錯誤可退，走綠界金流退刷（決策 #6）。
- **庫存**：不擋單，以「下單後訂製、交期至少 XX 天」於商品頁／結帳／說明頁告知（決策 #2）。

> 以下參數**上架前才需定案**，本 PRD 先佔位：戒指交期天數、各選項加價金額、金屬價格管理方式、首款戒指的具體寶石／金屬組合。

## 10. 關鍵假設與風險

| 項目 | 假設 | 風險 | 緩解 |
|---|---|---|---|
| 客群 | 獨立女性願意線上自助購買高客單彩寶戒指 | 高客單線上轉換可能偏低、需信任建立 | 呈現精緻＋規格透明＋多圖＋明確交期；先小批驗證 |

| 節奏 | 單人開發按里程碑序列可完成 | 時程拉長、易半途失焦 | 逐任務、做完停下檢核；對外承諾加 20–30% 緩衝 |
| 法規 | 客製品可主張七天例外 | 用詞不當有法律風險 | 結帳告知＋條款載明，**律師審定**後上線 |

## 11. 待解問題 / 開放決策（餵後續）

1. **首款戒指的具體設定**：哪種寶石（如藍寶／碧璽／彩色系）、哪些金屬色、戒圍範圍、可選數量上限？設計需**日常戴得住、耐看好搭**（非僅上場合的 statement piece）。→ 影響 P05 線框與 T43 種子資料。
2. **品牌與文案語氣**：品牌名、視覺基調、文案人稱與口吻（**quiet luxury／小眾設計師品牌調性**：低調、設計先行、「懂的人看得出」，不為 logo 溢價、不張揚）→ P05 前的 Brand Guide（`docs/brand-guide.md`）。
3. **價位帶內的成本結構**：在 NT$20k–50k 帶下，材質與寶石選擇如何兼顧質感與毛利？（上架前定）
